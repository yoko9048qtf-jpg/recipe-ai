## バージョニング方針

Semantic Versioning（`Major.Minor.Patch`）に準拠する。

| 種別 | 基準 | 例 |
|---|---|---|
| Patch | 不具合修正（`fix:`） | v1.0.0 → v1.0.1 |
| Minor | 機能追加・後方互換な変更（`feat:`） | v1.1.0 → v1.2.0 |
| Major | 互換性を壊す変更 | v1.x.x → v2.0.0 |

`docs:` / `refactor:` / `test:` / `chore:` のみのコミットはバージョンを更新しない。
